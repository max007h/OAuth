POC OAuth2 RAR+PAR — Autorisation Fine-Grained avec PingIdentity

Documentation technique — Stack self-hosted Docker | Spring Boot · Angular · PingFederate · PingAuthorize

1. Description du POC

Proof of Concept démontrant l'implémentation d'un système d'autorisation fine-grained dans un contexte bancaire, combinant OAuth2 RAR (Rich Authorization Requests — RFC 9396) et PAR (Pushed Authorization Requests — RFC 9126) avec la stack PingIdentity déployée en self-hosted via Docker.

L'objectif est de montrer comment encoder l'intention métier (IBAN, montant, type de virement) dans le token OAuth2 dès la phase d'autorisation, et de déléguer les décisions d'accès fines à un PDP (Policy Decision Point) centralisé.

Stack technique

Angular (SPA) Spring Boot Spring Security PingFederate (AS) PingAuthorize (PDP) PingDirectory (LDAP) Docker Compose OAuth2 RAR · PAR · PKCE

2. Scénarios du POC

# Acteur Rôle Scénario
1 Customer_1 CUSTOMER Consulter ses propres virements (accès limité à son IBAN)
2 Customer_2 CUSTOMER Consulter ses propres virements (accès limité à son IBAN)
3 Customer_1 / Customer_2 CUSTOMER Initier un virement / paiement via PAR+RAR (IBAN + montant encodés dans le token)
4 Approbateur_1 APPROVER Consulter les demandes de paiement en attente de validation
5 Approbateur_1 APPROVER Valider ou rejeter un virement au-delà du plafond autorisé
